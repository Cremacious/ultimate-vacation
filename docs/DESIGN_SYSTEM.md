# Design System Direction

This is not a final visual spec. It is the set of early design rules that keep the product coherent while we build.

## Design Goals

- premium but not pretentious
- playful but still functional
- minimal, airy, and easy to scan
- slightly glassy with restrained depth
- a little sassy in copy, never messy in layout

## Visual Thesis

Travel planning should feel like a polished metro control panel with color-pop confidence, playful attitude, and enough structure to keep the whole group moving.

## Aesthetic Rules

- use layered surfaces, not clutter
- prefer soft gradients, depth, and blur over heavy borders everywhere
- keep color count tight
- let typography and spacing do most of the work
- use cards only when they create real interaction clarity

## Color Direction

Approved direction:

- cool off-white and misty gray-blue base tones
- deep charcoal and slate for anchors
- bright cyan-blue as the primary action color
- hot pink as a secondary accent, not the main CTA
- punchy yellow for playful highlights
- overall palette should feel metro, poppy, and slightly 80s without going neon-chaotic

Avoid:

- default purple SaaS palettes
- overly sandy or beachy styling
- flat monochrome product surfaces
- harsh neon overload

## Typography Direction

- strong headline hierarchy
- clean sans-serif for product UI
- compact uppercase utility labels where helpful
- no overly decorative travel fonts

## Surface Strategy

- hero and marketing surfaces can be more atmospheric
- app surfaces should be calmer and more operational
- glass effects should be subtle and readable
- shadows should suggest depth, not fog
- key panels should feel stacked and tactile, not perfectly flat

## 3D Material Rules

- use bottom-edge ledges on important cards, panels, pills, and buttons
- 3D treatment should feel like a crisp physical layer, not skeuomorphic plastic
- primary buttons should have a visible darker bottom lip
- important cards can use a shallow ledge plus shadow for a more dimensional look
- inputs should stay cleaner than buttons, with softer depth cues
- reserve the strongest 3D effect for actions and high-importance surfaces

## Motion Direction

- soft entrance motion
- small hover lift on interactive surfaces
- phase transitions should feel smooth and directional
- motion should help orientation, not distract from tasks

## Copy Direction

### Brand tone

- smart
- cheeky
- capable
- reassuring
- stylish
- a little bossy in a fun way

### Copy rules

- headlines can have attitude
- product instructions must stay clear
- empty states should feel helpful and lightly witty
- avoid trying too hard to sound funny
- copy can gently roast disorganization, never the user
- the product should sound like the competent friend who already booked the table

## UX Rules

- show what matters now
- keep primary actions obvious
- reduce decision fatigue
- expose advanced features progressively
- make collaboration feel guided, not overwhelming

## Accessibility Principles

- maintain strong contrast even with translucent surfaces
- avoid using glass effects as a replacement for structure
- touch targets must be generous
- motion should remain subtle and tolerable
- important checklist states should never depend on color alone

## Design Decisions To Lock Soon

- final product name
- final font direction
- initial marketing CTA language
- premium lock visual language
- trip phase iconography style

## Approved Reference Direction

The current reference style is:

- metro base palette
- colorful 80s-adjacent accents using blue, pink, and yellow
- blue primary CTA
- playful sassy copy
- ledged 3D buttons and card bottoms
- cleaner product UI than the marketing mood board
